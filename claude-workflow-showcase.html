廣告報表自動化工作流

全自動化 · 領先時代 B.R.A.N.D · 每週兩次，手動啟動

輸入 /ads-report，Claude 自動判斷今天是週一還是週五，執行對應流程。從三個平台拉數據、交叉歸因、跑判定邏輯，到輸出 Excel 報表，一個指令完成。

週一大盤點

6 sheet Excel · 廣告歸因 × 四象限判定 · 諮詢品質分析

週五快檢

本週一～週四 · 只看 CPL 有無超標 · 純文字，快速決策

三路數據來源
  • Meta 廣告後台 — 花費、CPL、各廣告成效
  • Systeme 名單系統 — 名單資料 + UTM 參數
  • Notion 諮詢資料庫 — 諮詢預約狀態
設定方式
  • 跟 CC 說明工作流（使用 Wispr Flow）
  • 依照 CC 指示，提供 API Keys 等資訊
  • 提供廣告判定方法與報表需求
  • 進行小規模測試，確認後請 CC 記錄

社群貼文月排程表

內容規劃 · AI 共創 · IG · Threads

輸入 /social-plan，由社群行銷的虛擬員工接手，根據品牌定位與當月目標，規劃完整的月排程，直接輸出可用的表格版本。

輸出內容
  • 月曆格式排程表
  • 教育型 / 娛樂型 / 轉換型配比
  • 每篇主題 + 文案角度（Content Pillar）
  • IG Reels 腳本方向
Agent 的思維框架 先確認這篇內容「為誰服務」（吸引新受眾 vs. 維繫現有受眾）、貼文「目的」（不為發而發，內容要為目的服務），並以信任建立路徑思考內容序列，不只看單篇效果。

Speed Reader · 文字轉語音工具

工具開發 · Deepgram Aura TTS · Karaoke 同步 highlight

描述需求，Claude Code 直接串接 Deepgram Aura TTS API 寫出工具。輸入文字，生成語音的同時取得逐字時間軸，播放時像 Karaoke 一樣逐字 highlight，已讀文字變灰。

技術流程
📝 輸入文字
↓
🔊 Deepgram TTS 生成音訊
↓
⏱ STT 取得逐字時間軸
↓
✨ 播放 + Karaoke highlight
功能
  • 6 種 Aura 語音（男/女不同語氣風格）
  • 語速 0.75x–1.25x 可調
  • 進度條 · 可暫停繼續
  • STT 失敗自動 fallback

國外顧問市場研究 × DigiLev 品牌願景

研究 · 輸出轉化

分別用 Gemini 與 Claude Chat 的 Research 功能深度研究國外 Notion 顧問市場，再帶到 Claude Code 整合成好讀的品牌願景網頁。

為什麼這樣用？ Research 功能負責廣度與深度的資料蒐集，Claude Code 負責把原始研究轉化成「可以快速理解」的格式。工具各司其職，不用同一個地方全包。
Prompt 所包含的元素
  • 大背景：定調市場
  • 以當地語言深入權威＋社群
  • 結構化數據的具體內容
  • 研究量體（500 篇以上）
  • 報告結果（四個角度）
Research Prompt（節錄）
針對美國、歐洲、亞洲市場，進行「Notion 顧問業」的深度掃描（請把 AI 現在的趨勢也一同納入考量）。請以當地語言搜尋權威報告與本月當地社群討論。我需要結構化的數據：市場規模、CAGR、競品分析、以及在地消費者偏好。請綜合閱讀至少 500 篇以上的資料，並標註來源。結論從四個角度來回答：成長的、衰退的、新的機會、以及應捨棄的類別是什麼？

添加自己在書中讀到的框架，生成理想中的結果。使用 AI 時的思維框架很重要，用進 prompt 也會生成跟別人比較不同的內容。
