AI 員工 × 品牌記憶架構
類型：系統架構
系統架構

同時經營品牌、專案合作，身兼多種角色並需要執行不同類型的工作，每個任務需要的思維框架都不同。我把不同專業角色封裝成 Claude Code 的 Agent（虛擬員工），任務一來，對應員工自動到位，不用每次重新解釋背景，也不用手動切換模式。

02
廣告報表自動化工作流
類型：全自動化 · 領先時代 B.R.A.N.D 課程 · 每週兩次，手動
全自動化

輸入 /ads-report，Claude 自動判斷今天是週一還是週五，執行對應流程。從三個平台拉數據、交叉歸因、跑判定邏輯，到輸出 Excel 報表，一個指令完成。

兩種模式
模式週期輸出
週一大盤點上週完整報表6 sheet Excel · 廣告歸因 × 四象限判定
週五快檢本週一～週四純文字 · 只看 CPL 有無超標
三路數據來源
  • Meta 廣告後台 — 花費、CPL、各廣告成效
  • Systeme 名單系統 — 名單資料 + UTM 參數（廣告歸因用）
  • Notion 諮詢資料庫 — 諮詢預約狀態（成交、未出席等）
自動歸因邏輯
  • utm_content 有值 → 歸因到具體廣告 ID
  • utm_content 空，utm_source = meta/ig/fb → 歸因到 Meta 廣告
  • utm_source = convertkit → 電子報，不算廣告
  • 無標籤 → 來源不明
設定方式
  • 跟 CC 說明工作流（使用 Wispr Flow）
  • 依照 CC 的指示，拿到各數據來源的相關資訊（API Keys 等）
  • 提供廣告判定方法、希望報表涵蓋哪些內容
  • 設定完成後，進行小規模測試
  • 沒問題後，請 CC 紀錄（md、skill 等）
03
廣告痛點分析 × TA 報告
類型：AI 共創 · 內容轉化
AI 共創 內容轉化
  • 串接 Notion 諮詢 DB → 篩出成交者記錄 → 痛點歸因分析 → TA 分析報告
  • 最後手動廣告圖文產出
  • 未來待優化：讓廣告的痛點呈現，對齊課程內容（解方）
為什麼這樣做？ 廣告文案最難的不是寫作，是「知道受眾在痛什麼」。用真實諮詢資訊回推，比單純做 TA 側寫分析還要準確。
04
社群貼文月排程表
類型：內容規劃 · AI 共創 · 平台：IG · Threads
內容規劃 AI 共創

輸入 /social-plan，由社群行銷的虛擬員工接手，根據品牌定位與當月目標，規劃完整的月排程——包含主題方向、格式配比、每篇文案角度，直接輸出可用的表格版本。

輸出內容
  • 月曆格式排程表
  • 教育型 / 娛樂型 / 轉換型配比
  • 每篇主題 + 文案角度（COntent Pillar）
  • IG Reels 腳本方向
社群行銷 Agent 的思維框架： 先確認這篇內容「為誰服務」（吸引新受眾 vs. 維繫現有受眾）、貼文「目的」（不為發而發，內容要為目的服務），並以信任建立路徑思考內容序列，不只看單篇效果。

06
Speed Reader · 文字轉語音工具
類型：工具開發 · API：Deepgram Aura TTS · Karaoke 同步 highlight
工具開發

描述需求，Claude Code 直接串接 Deepgram Aura TTS API 寫出工具。輸入文字，生成語音的同時取得逐字時間軸，播放時像 Karaoke 一樣逐字 highlight，已讀文字變灰，方便跟讀或校對。

技術流程
輸入文字 → Deepgram TTS 生成音訊 → STT 取得逐字時間軸 → 播放 + Karaoke highlight
功能
  • 6 種 Aura 語音（男/女不同語氣風格，進 Deepgram 帳號可查看語音類型）
  • 語速 0.75x–1.25x 可調
  • 進度條 · 可暫停繼續
  • 分頁標題顯示進度
  • STT 失敗自動 fallback（均勻分配時間軸）

07
國外顧問市場研究 × DigiLev 品牌願景
類型：研究 · 輸出轉化
研究 輸出轉化

分別用 Gemini 與 Claude Chat 的 Research 功能深度研究國外 Notion 顧問市場，產出各自的報告與洞察，再把兩份研究結果帶到 Claude Code，整合成結構化、適合閱讀的品牌願景網頁。

為什麼這樣用？

Research 功能負責廣度與深度的資料蒐集，Claude Code 負責把原始研究結果轉化成「可以快速理解」的網頁格式。工具各司其職，不用同一個地方全包。

Research 模式使用的 Prompt
針對美國、歐洲、亞洲市場，進行「Notion 顧問業」的深度掃描（請把 AI 現在的趨勢也一同納入考量）。請以當地語言（英語、日語等）搜尋權威報告與本月當地社群討論。我需要結構化的數據：市場規模、CAGR、競品分析、以及在地消費者偏好。請綜合閱讀至少 500 篇以上的資料，並標註來源。結論從四個不同的角度來回答，成長的、衰退的、新的機會、以及應捨棄的類別是什麼，並解釋為什麼？

Prompt 所包含的元素：

  • 大背景：定調市場
  • 以當地語言深入權威＋社群
  • 結構化數據的具體內容
  • 研究量體
  • 報告結果
接著分析：如果要在台灣建立一間足以壟斷市場的數位工作流設計顧問公司，幫助大中小企業用不同的數位工具（先以 Notion 為主，後面可以延伸整合不同工具）製作內部工作流營運系統，實際上可落地執行的藍圖跟個階段運作策略為何？備註：壟斷的概念來自 Peter Thiel，重點在於建立一個根本不用競爭、跟其他相同性質的競品拉開距離的地位。另外，在市場供需概念方面，可以參考 Oversubscribed 一書中的概念。

添加自己在書中讀到的框架，生成理想中的結果。（使用 AI 時的思維框架很重要，用進 prompt 也會生成跟別人比較不同的內容）
